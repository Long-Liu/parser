# Excel 各Sheet尾部特征分析

基于 `excel/xxx项目主体施工动态成本表-样式 - 副本.xlsx` 分析。

## 汇总表

| 停止规则类型 | 特征 | 触发Sheet |
|------------|------|----------|
| **A: 注释行** | A列以"注："开头 | 表8 (Row 18) |
| **B: 新子表标志** | 数据行之后出现"金额单位"等新表头标记 | 毛利 (Row 13) |
| **C: 尾部说明** | 数据结束后出现无层级编号的说明/计算文字 | 表9-2 (Row 27-32) |
| **D: 连续空行** | 数据结束后大段空行 | 表5 (Row 11+)，表6 (Row 22+) |
| **E: 自然结束** | 数据到最后一行自然结束，无尾部噪音 | 其余多数Sheet |

## 逐Sheet详情

### 1. 毛利
- **特征**: Sheet包含**两个独立子表**
  - 子表1：Row 1-10（经济考核指标及预计完工量指标毛利、毛利率）
  - 子表2：Row 13-18（另一个维度的毛利表）
- **停止规则**:
  - 子表1的数据结束于Row 10（Row 11-12空行）
  - Row 13 `金额单位：万元` + Row 14 `序号` 为新子表起始
- **触发**: B类 — A列匹配 `^金额单位`

### 2. 动态指标
- **特征**: Row 3-93全部为有效数据，包含汇总行（七、八、九）
- **停止规则**: 自然结束
- **触发**: E类

### 3. 表1 人工费-动态
- **特征**: Row 5-77，Row 76为 `二、专项人工费`，Row 77为 `合计`
- **停止规则**: 自然结束（合计行也是有效数据）
- **触发**: E类

### 4. 表1-1人工费动态
- **特征**: 仅3行（1表头行+1子表头行+1数据行）
- **停止规则**: 自然结束
- **触发**: E类

### 5. 表2 社会统筹-动态
- **特征**: 结构与表1一致
- **停止规则**: 自然结束
- **触发**: E类

### 6. 表3 现场管理费-动态
- **特征**: Row 5-85，最后数据Row 85 `18.8 塑料凳`
- **停止规则**: 自然结束
- **触发**: E类

### 7. 表4 机械费用-动态
- **特征**: Row 2-40，Row 40为 `小计`
- **停止规则**: 自然结束
- **触发**: E类

### 8. 表5建安标段对比分析表
- **特征**: 实际数据仅10行（Row 1-10），但Sheet声明max_row=65493（大量空行）
- **停止规则**: 数据行后出现**连续N行全空**
- **触发**: D类

### 9. 表6 建筑工程-动态
- **特征**: Row 1-19数据，Row 19为`小计（不含税）`和`税率`，之后Row 20-24有空行
- **停止规则**: 连续空行
- **触发**: D类

### 10. 表7 安装工程-动态
- **特征**: Row 1-30有数据，Row 30为 `合计`，之后Row 31-40有零星内容或空行
- **停止规则**: 自然结束或连续空行
- **触发**: E/D类

### 11. 表8 其他项目清单与计价汇总表
- **特征**: Row 17为 `合计`，Row 18为 **`注：本表合计金额汇入报价表一工程项目投标总价表。`**
- **停止规则**: A列匹配 `^注：`
- **触发**: A类

### 12. 表9建安材料成本表
- **特征**: Row 3-49，Row 49为 `总计`
- **停止规则**: 自然结束
- **触发**: E类

### 13. 表9-1 混凝土管理台账
- **特征**: 526行全部为台账明细数据
- **停止规则**: 自然结束
- **触发**: E类

### 14. 表9-2 钢筋管理台账
- **特征**: Row 1-26为有效数据，Row 27起出现尾部说明：
  - Row 27: `更新2025/8/2`
  - Row 28: `图算量合计Kg`
  - Row 29: `钢筋根数`
  - Row 30: `20≦连接量4%`
  - Row 31: `措施筋1%`
  - Row 32: `合计Kg`
- **停止规则**: A列和B列为空的行序列开始（表9-2中尾部说明行A列和B列均为空，仅C列有文字）
- **触发**: C类

### 15. 表9-3 安装材料（回归主合同清单）
- **特征**: 502行有效数据，max_row=888（大量尾部空行）
- **停止规则**: 连续空行
- **触发**: D类

## 统一停止规则设计

综合以上分析，建议采用**多级fallback**策略：

```yaml
stop_rules:
  # 规则1：注释行 — 命中立即停止
  - type: "cell_match"
    patterns:
      - "^注："              # 如 表8 Row 18
      - "^注 :"
      - "^说明："            # 预留
    columns: ["A"]           # 在A列匹配
    action: "stop"

  # 规则2：新表开始标志 — A列出现这些值，停止当前解析
  - type: "cell_match"
    patterns:
      - "^金额单位"          # 如 毛利 Row 13
    columns: ["A"]
    action: "stop"

  # 规则3：尾部噪音检测 — A列和B列都为空但C列有内容
  # （数据行通常A列有层级号或B列有名称，尾部说明行两者都空）
  - type: "consecutive_empty_columns"
    empty_columns: ["A", "B"]
    consecutive_rows: 2
    description: "连续2行A列和B列均为空（非汇总行特征）"
    action: "stop"
    # 注意：这条规则仅在特定sheet启用，因为表9-2才有此特征

  # 规则4：兜底 — 连续N行全空
  - type: "consecutive_empty_rows"
    count: 5
    description: "连续5行所有列为空时停止"
    action: "stop"
```

其中规则1、2、4是**全局默认规则**，规则3是**按模板可选的规则**。

> 注意："合计"、"小计"、"总计" 行**不作为停止规则**，它们都是有效数据需要入库。
